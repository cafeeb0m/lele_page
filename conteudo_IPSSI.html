ISO x TCP/IP

ISO - Características Gerais

ISO (Organização Internacional de Normalização) é um conjunto de normas e estruturas que visa a efetividade e encorajamento do comércio de bens e serviços. No total, há 91 países participantes da ISO, em especial, os EUA, que possui como representante na ISO a ANSI (Instituto Nacional Americano de Padrões) e, também, o Brasil, possuindo a NBR. As 5 principais ISO correspondentes às normas e padrão de qualidade são: ISO 9000, ISO 9001, ISO 9002, ISO 9003 e ISO 9004. A ISO 9000 é a principal, servindo como base para as demais citadas, mas, ainda que seja o pilar das outras, não é a mais abrangente, este papel fica para 9001 que possui todos os 20 elementos de normas qualidades, em seguida da 9002, que possui 18 e, por fim, a 9003 contendo apenas 12.

Camadas de Protocolo - ISO

A ISO, na década de 70, criou um modelo de protocolo de redes batizado de OSI (Interconexão de Sistemas Abertos), sua funcionalidade é estabelecer regras de comunicação entre múltiplas máquinas. A OSI é composta por 7 camadas de protocolos, sendo elas:

  • Camada - Física: Envolve as entradas e processos que os dados percorrem para alcançar seu destino. Como sugere o nome, o caminho é por meios físicos, como dispositivos, cabos de redes, etc.
  • Camada – Enlace ou Ligação: Responsável pela análise dos dados enviados da primeira camada, verificando se há erros e corrigindo caso haja. Parte essencial da ISO, pois possibilita a boa execução eficiente e sem erros das camadas superiores. Possui duas subcamadas: MAC e LLC.
  • Camada – Rede: Funciona como um seletor de mensagens, decidindo quais mensagens mais importantes a serem enviadas primeiro e definir a melhor rota possível para elas, aqui se encontra o protocolo IP (Endereço da Máquina). Monitora os envios do endereço MAC para o destinatário IP, como são vários trajetos de dados feitos até o IP, o MAC está em constante alteração, enquanto que o IP se mantém sempre fixo.
  • Camada – Transporte: Os “caminhões” de dados, ela garante o êxito dos dados e o recebimento dos dados da terceira camada. Os protocolos mais comuns aqui são o UDP e TCP, o primeiro sendo mais utilizado e mais rápido que o segundo devido sua garantia da entrega da mensagem.
  • Camada – Sessão: Responsável por estabelecer a conexão entre os hosts, iniciando e encerrando essa conexão. Também oferece suporte para essas sessões como registro de log e tarefas de seguranças.
  • Camada – Apresentação: Basicamente funciona como um “tradutor” dos dados enviados para serem utilizados na última camada, também criptografa os dados caso necessite.
  • Camada – Aplicação: Camada de consumo de dados, aqui tem a interação direta entre homem e máquinas, protocolos conhecidos como o HTTP e o FTP estão presentes aqui.

TCP/IP - Características Gerais

O conjunto de protocolos TCP/IP é de extrema importância para o funcionamento da internet como um todo. Surgiu no final da década de 60 nos EUA junto ao projeto ARPANET, projeto esse que envolvia a comunicação entre múltiplos computadores, ou seja, o nascimento da internet. Anos mais tarde, após o fim da guerra fria e de uma possível guerra nuclear, surge o TCP/IP em 1975, estabelecendo conexão da universidade de Stanford, EUA, com a universidade College London, na Inglaterra. Ao longo do tempo, nasceu o protocolo HTTP e estabeleceu a internet como é conhecida hoje em dia. O conjunto TCP/IP é dividido em camadas para melhor gerenciamento e segurança de dados, garantindo que a transmissão de dados seja padronizada e independente dos dispositivos usados nos processos, não a tornando dependente de linguagem de programação, sistema operacional ou arquitetura. A primeira camada é a de aplicação, envolve a seleção dos protocolos que serão usados pela TCP/IP para iniciar a transmissão de dados, utilizando de HTTP e HTTPS a partir de URL, por exemplo. Já na segunda camada se encontra o transporte dos dados, fazendo uso de portas TCP para inicializar a transferência, essas portas são usadas em conjuntos com o IP. A terceira camada envolve o IP que, basicamente, envia um sinal afirmativo para realizar o envio do pacote de dados, lidando com as interfaces dos hosts e transformando os dados em datagramas. E por fim, temos a camada de interface, que lida diretamente com a transferência de dados em si entre os hosts.

Rede Enlace

A rede de enlace (ou ligação) é responsável pela análise de qualidade e controle de fluxo dos dados enviados da camada física. Aqui se encontra tecnologias como as VLans e Token ring, além disso. Dispositivos switches funcionam aqui. A rede se divide em duas subcamadas: camada MAC e camada LLC.

  • MAC: Camada responsável pela conexão de vários computadores em uma rede. Toda máquina possui um endereço físico por rede, no caso, o endereço MAC, é a partir desse endereço que a camada utiliza para enviar os dados corretamente.
  • LLC: Responsável pelo controle de fluxo dos dados, garante que múltiplos dados convivam dentro de uma rede.
